## Overview

The PurchaseItem component provides a dedicated interface for users to complete item purchases. It displays comprehensive item details, pickup location and time information, and payment options through CashApp and PayPal.

## Location

`src/PurchaseItem.js`

## Props

<ParamField path="item" type="object" required>
  The selected item object passed from the Feed component, containing:
  - `item_name` - Name of the item
  - `item_description` - Description text
  - `image_url` - URL to the item image
  - `price` - Item price
</ParamField>

## Implementation Example

```javascript
import { useState } from 'react';
import Feed from './Feed';
import PurchaseItem from './PurchaseItem';
import { Routes, Route } from 'react-router-dom';

function App() {
  const [selectedItem, setSelectedItem] = useState(null);
  
  return (
    <Routes>
      <Route path="/feed" element={
        <Feed modifyItem={setSelectedItem} />
      } />
      <Route path="/purchase-item" element={
        <PurchaseItem item={selectedItem} />
      } />
    </Routes>
  );
}
```

## Data Flow

1. User browses items in the Feed component
2. User clicks "Buy Now" on an item
3. Feed calls `modifyItem(items[index])`
4. Feed navigates to `/purchase-item`
5. PurchaseItem receives item via props
6. PurchaseItem displays item details and payment options

```javascript
console.log(item);
```

The component logs the item object on render for debugging purposes.

## Current Limitations

<Warning>
Several features need implementation for production use:

1. **Payment Integration**: Payment buttons have no functionality
2. **Dynamic Seller Info**: Seller name is hardcoded as "Tommy T"
3. **Dynamic Pickup Details**: Location and time are hardcoded
4. **Transaction Processing**: No backend integration for completing purchases
5. **Error Handling**: No validation or error states
6. **Loading States**: No feedback during payment processing
</Warning>

## Potential Improvements

<Tip>
To make this component production-ready:

- Integrate with CashApp and PayPal payment APIs
- Fetch dynamic seller information from user data
- Allow sellers to set pickup locations and times
- Implement transaction recording in the database
- Add confirmation dialogs before purchase
- Include order tracking functionality
- Add ability to contact seller from purchase page
</Tip>
