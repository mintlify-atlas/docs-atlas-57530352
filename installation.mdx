## Installation Steps

### 1. Clone the Repository

Clone the Trojan Marketplace repository to your local machine:

```bash
git clone <repository-url>
cd csci201-final-project
```

<Info>
  The project name in `package.json:2` is `csci201-final-project` with version `0.1.0`.
</Info>

## Backend Setup

The Trojan Marketplace frontend requires a Java servlet backend for full functionality.

<Steps>
  <Step title="Backend Requirements">
    The application expects the backend server at:
    ```
    http://localhost:8080/Final_Project/
    ```
    
    Ensure your Java servlet container (Tomcat, etc.) is configured to run on port 8080.
  </Step>

  <Step title="Required Servlets">
    The following servlets must be implemented and accessible:
    
    **LoginServlet** - Used in `src/Login.js:80`
    ```
    POST http://localhost:8080/Final_Project/LoginServlet
    ```
    Handles user authentication with username/password
    
    **ItemFetchServlet** - Used in `src/Feed.js:15`
    ```
    GET http://localhost:8080/Final_Project/ItemFetchServlet
    ```
    Returns array of marketplace items with properties:
    - `item_name` - Name of the item
    - `item_description` - Description text
    - `image_url` - URL to item image
  </Step>

  <Step title="CORS Configuration">
    Configure CORS headers on your backend to allow requests from `http://localhost:3000`:
    
    ```java
    response.setHeader("Access-Control-Allow-Origin", "http://localhost:3000");
    response.setHeader("Access-Control-Allow-Methods", "GET, POST");
    response.setHeader("Access-Control-Allow-Headers", "Content-Type");
    ```
  </Step>
</Steps>

<Warning>
  Without the backend running, you'll encounter connection errors when attempting to log in or view the marketplace feed.
</Warning>

## Troubleshooting Installation

<AccordionGroup>
  <Accordion title="npm install fails with permission errors">
    If you encounter permission errors during installation:
    
    **On macOS/Linux:**
    ```bash
    sudo chown -R $USER:$(id -gn $USER) ~/.npm
    sudo chown -R $USER:$(id -gn $USER) ~/.config
    ```
    
    **On Windows:**
    Run your terminal as Administrator
  </Accordion>

  <Accordion title="Dependencies have vulnerabilities">
    If npm audit shows vulnerabilities:
    
    ```bash
    npm audit fix
    ```
    
    For breaking changes:
    ```bash
    npm audit fix --force
    ```
    
    <Warning>
      Be cautious with `--force` as it may introduce breaking changes.
    </Warning>
  </Accordion>

  <Accordion title="Port 3000 already in use">
    If port 3000 is occupied:
    
    **Option 1**: Use a different port
    ```bash
    PORT=3001 npm start
    ```
    
    **Option 2**: Kill the process using port 3000
    ```bash
    # On macOS/Linux
    lsof -ti:3000 | xargs kill -9
    
    # On Windows
    netstat -ano | findstr :3000
    taskkill /PID <PID> /F
    ```
  </Accordion>

  <Accordion title="Styled Components not working">
    If styled components aren't rendering properly:
    
    1. Verify installation: `npm list styled-components`
    2. Clear node_modules and reinstall:
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
    3. Clear browser cache and reload
  </Accordion>

  <Accordion title="Backend connection fails">
    If you cannot connect to the backend:
    
    1. Verify backend is running on port 8080
    2. Check CORS configuration allows `http://localhost:3000`
    3. Test servlets directly:
    ```bash
    curl http://localhost:8080/Final_Project/ItemFetchServlet
    ```
    4. Check browser console for specific error messages
  </Accordion>
</AccordionGroup>
