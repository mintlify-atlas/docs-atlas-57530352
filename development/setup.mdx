## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** (v14 or higher)
- **npm** (comes with Node.js)
- **Java Development Kit** (JDK 8 or higher) for backend
- **Apache Tomcat** (v9.0 or higher) for servlet container
- A code editor (VS Code, IntelliJ IDEA, etc.)

## Project Structure

The Trojan Marketplace consists of two main parts:

```
trojan-marketplace/
├── source/                  # React frontend
│   ├── public/
│   ├── src/
│   │   ├── components/
│   │   ├── App.js
│   │   ├── Main.js
│   │   └── index.js
│   └── package.json
└── backend/                 # Java servlets
    └── Final_Project/
```

## Frontend Setup

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:
    ```bash
    git clone <repository-url>
    cd source
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required npm packages:
    ```bash
    npm install
    ```
    
    This will install the following key dependencies from `package.json:5-15`:
    - **react** (^18.2.0) - Core React library
    - **react-dom** (^18.2.0) - React DOM rendering
    - **react-router-dom** (^6.10.0) - Client-side routing
    - **styled-components** (^5.3.9) - CSS-in-JS styling
    - **axios** (^1.3.6) - HTTP client for API calls
    - **react-scripts** (5.0.1) - Build tooling
  </Step>

  <Step title="Start the development server">
    Run the development server:
    ```bash
    npm start
    ```
    
    The application will start at `http://localhost:3000` and automatically open in your browser.
  </Step>
</Steps>

<Note>
  The development server supports hot reloading - changes to source files will automatically refresh the browser.
</Note>

## Backend Setup

<Steps>
  <Step title="Configure Tomcat">
    Set up Apache Tomcat server:
    - Download and install Tomcat 9.0+
    - Note the installation directory
    - Configure Tomcat port (default: 8080)
  </Step>

  <Step title="Deploy servlets">
    The backend expects servlets at:
    ```
    http://localhost:8080/Final_Project/
    ```
    
    Key servlets used by the frontend:
    - `LoginServlet` - User authentication (Login.js:79-80)
    - `CreateAccountServlet` - Account creation (CreateAccount.js:85-86)
    - `ItemFetchServlet` - Retrieve marketplace items (Feed.js:14-15)
    - `AddItemServlet` - Create new listings (CreateSale.js:24-25)
  </Step>

  <Step title="Start the backend server">
    Start Tomcat to enable backend functionality:
    ```bash
    # From Tomcat bin directory
    ./catalina.sh run    # macOS/Linux
    catalina.bat run     # Windows
    ```
  </Step>
</Steps>

<Warning>
  The frontend expects the backend to be running at `http://localhost:8080/Final_Project/`. Ensure this URL matches your Tomcat configuration.
</Warning>

## Database Configuration

The backend servlets require database connectivity:

1. Set up a MySQL/PostgreSQL database
2. Configure connection parameters in servlet configuration
3. Ensure database tables are created for:
   - User accounts
   - Item listings
   - Transactions
   - Messages

## Available Scripts

From `package.json:17-21`, the following npm scripts are available:

### `npm start`
Runs the app in development mode at [http://localhost:3000](http://localhost:3000).

### `npm build`
Builds the app for production to the `build` folder:
```bash
npm run build
```

Optimizes the build for best performance with minified bundles.

### `npm test`
Launches the test runner in interactive watch mode:
```bash
npm test
```

### `npm run eject`
<Warning>
  This is a one-way operation. Once you eject, you can't go back!
</Warning>

## Common Issues

### Port Conflicts
If port 3000 is in use:
```bash
# Set custom port
PORT=3001 npm start
```

### CORS Errors
If you see CORS errors with the backend, ensure Tomcat is configured to allow requests from `http://localhost:3000`.

### Module Not Found
If you encounter missing module errors:
```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```
